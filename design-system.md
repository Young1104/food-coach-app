# 开发交付规格 · Food Coach AI 营养教练

**对应原型**：`FoodCoach-AI营养教练-高保真原型.html`
**范围**：PRD 中 P0 杀手锏流程（对话式输入 → 生成 → 今日计划+为什么 → 吐槽式调整）
**视觉语言（最新·多色阶绿卡系统）**：中性底 + **三档绿色填充卡**建立层次与重点（参考 Testora / 金融卡组）。

色阶与卡片角色：
| 角色 | 色值 | 用途 |
|---|---|---|
| 清新浅绿（主卡） | `#DCEFC8` 底 + 深绿字 `#16431F` | 核心指标卡（今日摄入），更"健康饮食"；含 `#1A7D4B` 进度环 + **白色宏量瓦片**浮于其上 |
| 主绿（按钮/激活） | `#1A7D4B` | 主 CTA、进度、底部 AI 方块 |
| 浅绿（强调卡/chips） | `#DCEFC8` 底 + 深绿字 `#16431F` | 强调型 feed 卡、标签 |
| 白卡（内容） | `#FFFFFF` + 1px 描边 | 列表、计划、图表等常规内容 |
| 亮黄绿点缀 | `#B8E986` | 深绿卡上的进度环/图标高光 |

要点：层次靠"深绿重卡 / 白内容"**填充对比**实现，背景保持中性；卡片扁平（描边、无大阴影）。

**绿色克制规则（重要）**：绿色**只允许出现在 4 处**——① 深绿主卡 `#103D2D`；② 主 CTA 按钮 `#1A7D4B`（采用/加入/下一步/生成）；③ 底部 AI 方块 + 激活导航图标；④ 进度环/进度条。其余一律中性：状态徽章=灰底灰字、`进行中`=墨底白字；做菜/为什么=白底描边墨字；区块"更多"/foot 链接=灰；科普卡=白。**禁止**把绿色撒在零散链接/小按钮上。

**按钮形状统一**：所有**按钮**（主 CTA + 小动作"做菜/为什么"）= 圆角矩形 `10px`；**徽章 / 筛选 chip / 选择 chip** = 药丸 `pill`。二者按组件类型区分，不混用。`button:focus{outline:none}` 移除浏览器默认（橙色）描边。

**对齐网格**：卡内所有元素左对齐到卡片内边距 `16px`（缩略图、动作按钮、为什么展开同一左缘）；右侧数值列固定最小宽度对齐。

**（历史）中性底 + 单绿主卡**（已升级为上面的三档色阶）：
- **图底/层次靠卡片对比，不靠背景刷色**：页面底色中性 `#F4F5F2`；**核心指标用一张绿色填充主卡（`#1A7D4B` 底 + 白字反色）**作锚点（如「今日摄入」），其余为白色卡 → 绿/白交替形成层次与重点。
- **卡片扁平**：白卡 = 1px 描边 `#E7E8E4` + **无阴影**（去掉大阴影）；绿主卡无边纯填充。仅浮动导航保留极轻阴影。
- **绿主卡内部反色细节**：进度轨 `rgba(#fff,.26)`/进度填充白；宏量做成 `rgba(#fff,.13)` 小瓦片白字；分隔线/链接白。
- **强调色**：深绿 `#1A7D4B`；近黑文字 `#1C1C1E` + 灰阶。
- **图标**：功能图标线性 SVG；emoji 仅作食物图片占位。
- **圆角**：卡片 16、**按钮 10**、中心 AI 方块 16；chips pill。
- **按钮可读性**：主按钮字重 700、足够对比（绿底白字 / 墨底白字 / 浅绿底绿字）；小操作按钮（为什么/做菜）字重 700、墨色文字。
- **底部导航**：白色浮动药丸；中心 AI = 绿色圆角方块 + 聊天图标 +「AI 助手」标签，等宽对齐。
- **字阶**：分区标题 19/800、卡片标题 15/600、正文 14、说明 12 muted。
**建议技术栈**：React + MUI(Material 3) / Jetpack Compose / Flutter（移动优先，单列布局）
**onboarding 原则**：**无介绍页、无强制注册**。App 打开即首页主功能（已渲染今日推荐计划）；个性化收集做成首页可关闭引导卡 + 全程「跳过」，价值先行、渐进采集。

**经设计审查的关键约定**：
- **语义一致**：进度卡是「今日计划」摄入（非已摄入）；进度环与「已吃 X/N 餐」联动，二者必须同源，不得各说各话。
- **每餐闭环**：餐卡含「✓ 吃过了」操作 → 标记完成 → 进度环 + 文案 + 卡片左边框状态同步更新（这是「每日指南」类产品的核心闭环，不可省）。
- **去假交互**：列表项若不可跳转，不放 chevron；同一卡片仅保留单一「为什么」展开入口；卡路里每餐只显示一次。
- **安全常驻**：首页底部常驻「AI 生成 · 不构成医疗建议」（PRD P0-5）。

## 概览

新手用户用最少输入（3 个对话问题 + 一句生活）得到一份带「为什么」解释的当日饮食计划，并可用自然语言实时调整。所有界面无术语裸露，强调「被引导」而非「自己研究」。

## AI-native 核心：全局 Co-pilot（最重要）

AI 不是某个页面里的输入框，而是**贯穿全局的中枢**。底部导航中心 = 黑色圆形 **Co-pilot** 入口（带青柠光环 ✦），在首页/食谱/进度/我的任意页面一键唤起。

- **一个入口收敛所有 AI 能力**：生成今天计划、调整某一餐、拍照记录、营养答疑，全部在 Co-pilot 对话内完成——不再散落成「首页底部输入框 + 生成按钮 + 定制链接 + 独立相机」四条路径。
- **Co-pilot 形态**：全屏对话（顶栏 ✦ + 关闭）+ 消息气泡 + 快捷意图 chip（生成/换一餐/拍照/问营养）+ 底部 composer（📷 + 输入 + 发送）。**输入框只存在于 Co-pilot**，绝不放在内容 feed 里。
- **意图驱动**：对话识别意图后直接操作计划并自动回到对应页面查看（如「换午餐」→ 改首页计划 → 回首页）。
- 首页仅保留一个轻量「✦ 问 AI 调整」入口指向 Co-pilot，避免重复 CTA。

## 首页 = 单一主任务「确定今日饮食计划」

首页的唯一核心 JTBD：**用户进来先把今天吃什么定下来**，其余都是次要。据此排布主次与状态机：

- **主任务状态机**：`AI 推荐` →（点「**采用今日计划**」）→ `进行中`（出现勾选框，吃完打勾，营养条联动）。状态用 planChip（推荐=浅绿 / 进行中=黑）+ 卡片底部 foot 切换体现。
- **层级**：① 今日计划（主，分组卡 + 主 CTA）② 今日营养（辅，紧凑卡：已摄入/目标条 + 宏量）。营养是计划的「结果」，永远在计划之后、视觉更轻。
- **不再有「今日推荐 vs 今日计划」两套**：统一为「今日计划」，推荐只是它的初始状态。
- **删除冗余**：去掉首页「今天/本周」分段（与周历重复）；本周趋势归并到「进度」页。
- **去卡中卡**：三餐 = 一张分组卡内的行 + 分隔线（非每餐一卡）；「为什么」= 行内单一展开链接（浅灰文字块，无嵌套彩色卡、无重复按钮）。
- **收敛行动点**：首页主行动只有一个（采用 / 打勾）；所有 AI 操作走底部 Co-pilot；移除标题处重复的「问 AI 调整」。

## 数字与文案一致性规范

- **单一数据源**：今日卡的热量、宏量、进度环、目标全部由三餐数据**实时计算**（`updateNutri` 汇总未删除餐次为目标、已勾选餐次为已摄入），任何一处不得硬编码独立数字。`目标 = Σ 餐次热量`（默认 1,480 = 380+560+540），跳过早餐后目标与宏量目标自动下调。
- **摄入与宏量同源**：未吃时显示 `已摄入 0 / 1,480 kcal` 且宏量 `0 / 172 g`（不再出现"摄入 0 却有宏量"的矛盾）；勾选后两者同步增加。
- **宏量与热量自洽**：每餐宏量按其热量配比设定（碳水/蛋白×4、脂肪×9 大致对得上），全日宏量目标 172/90/42 g。
- **时间一致**：状态栏 9:41 → 问候「早上好」。
- **指标表达清晰**：进度页两格改为含义明确且互不矛盾的「本周日均 1,460 kcal」「达标天数 5/7 天」；连续天数(4)/已记录(28)/目标达成(40%) 在首页·进度·我的三处保持一致。

## 交互一致性规范（审查后）

- **导航采用历史栈**：tab 切换重置栈根；详情/相机/Co-pilot/问卷为叠加层入栈；返回严格逐层出栈（如 Co-pilot→相机→返回必回到 Co-pilot，不跳首页）。
- **详情页数据驱动**：所有食物（计划三餐 + 配方库）走同一数据源，详情的图/名/热量/宏量必须与来源卡一致，禁止 id 复用导致内容错位。
- **详情主按钮按来源切换**：来自今日计划 → 「✦ 换一餐」（唤起 Co-pilot）；来自配方库 → 「加入今日计划」（确认后返回）。
- **相机每次进入复位**：清除上次识别结果卡。
- **营养语义分层**：热量为唯一动态追踪指标（已摄入/目标，随打勾变化）；宏量为「今日计划营养构成」（静态克数），两者标签明确区分，不混用。
- **可点元素必有响应**：周历日期可选中；无目的地的箭头/affordance 一律移除或给出反馈。
- **对比度**：底部导航未选标签由 `muted-2` 调深为 `muted`，满足可读。

## 信息架构（底部导航 · 最新）

5 项：**首页 · 食谱 · ✦(Co-pilot) · 进度 · 我的**。中心为黑色圆形 Co-pilot。
- **首页 = 今日计划(铺开) + 进度概览 + 科普**：顶部「今日摄入」卡(热量进度条 + 碳水/蛋白/脂肪 + 连续天数，合并进度);其下「今日计划」三餐直接铺开(采用/打勾/为什么/做菜);再下健康科普 feeds。用户进首页即可完成核心动作，无需跳转。
- 「计划」独立 tab 取消(铺开进首页)；**进度** tab 保留为深度 Insights(总进度/趋势/连续)，首页只放快照。
- **科普 feeds**：每张卡点击 → 唤起 Co-pilot 并自动提问，AI 即时讲解(无独立文章死页)。
- 所有功能图标为线性 SVG sprite(`<symbol>`+`<use>`)；emoji 仅作食物图片占位。

## AI 视频通话教做菜（signature，来自原 PRD）

闭环：**指定菜谱 → 材料包送达 → AI 视频通话教做菜**。
- **入口**：食物详情页显示「✅ 材料包已送达」+ 主按钮「🍳 AI 视频教我做」；今日计划每餐也有「🍳 做菜」。
- **通话页 `#cooking`**（深色，视频通话美学）：
  - 顶栏：最小化 + 标题 + 「● 通话中 计时器」（每秒走动）。
  - 教练区：AI 教练头像 + **脉冲说话光环** + **语音波形动画** + 底部**实时字幕气幕**；右上**自拍 PIP**。
  - 步骤底卡：「第 N 步 / 共 M 步」+ 步进点 + 步骤标题/说明，随「下一步」推进、字幕同步更新。
  - 控制条：🎤 静音 / 「下一步 ⏭」（主，末步变「完成 ✓」）/ 📞 结束（红）。
  - 结束 → 回「计划」页 + 该餐自动记录 toast。
- **状态机**：step 0..M-1；末步「完成」触发 cookEnd(done=true)。
- **状态栏配色**：深色页（cooking/camera）状态栏文字转白（`.statusbar.lt`）。

## 信息架构（旧·保留说明）

- **首页 Home**：今天/本周分段（黑色高对比）；青柠 hero 卡含周日历条 + **半圆热量仪表**（已摄入/目标/已消耗，随勾选实时填充）；碳水/蛋白/脂肪**彩色宏量瓷砖**（蓝/青柠/桃）；「✦ 生成我的一天」入口；今日推荐餐卡（缩略图进详情、☑ 标记吃过、「为什么」展开）；「和 AI 说一句」调整区；安全声明常驻。本周视图含 7 日柱状趋势 + 日均/达标天数。
- **食谱 Recipes**：搜索框 + 分类 chip（黑色激活）+ 双列食谱卡（图/名/热量·时长），点击进详情。
- **拍照记录 Camera**：深色取景器 + 四角青柠定位框 + 快门；点快门弹出识别结果卡（菜名/热量/置信度/记一笔）。
- **进度 Progress**：总进度青柠卡（黑色进度条 + 百分比药丸）；营养(桃)/运动(蓝)双瓷砖；深色「连续 N 天」卡；你的目标卡；本周摄入柱状图。
- **我的 Profile**：头像 + 统计三栏（连续/已记录/达成）+ 设置列表（目标/忌口/提醒/账号/帮助），目标与忌口直达问卷复用。
- **食物详情 Detail**：青柠圆角 hero + 大图；标题 + 四宫格宏量；份量/规格；加入餐次按钮 + 餐次下拉；**为什么推荐**；营养成分明细。

## 视觉规范（减法后）

- **黑色作为强调色**：分段控件、进度条、Co-pilot FAB、仪表弧、分类激活 —— 黑 + 青柠高对比是高级感主来源。
- **圆角收敛**：统一刻度 `xs10 / 14 / 16 / lg20 / full`；卡片 16、hero 20，**不超过 20px**（旧版 24–30px 过圆已修正）。
- **间距统一**：屏幕边距 18、卡内 16、卡间 12、分区标题上 22。修掉了不一致的内边距。
- **卡片层级合并**：首页 hero 一张卡内含「周历 + 半圆仪表 + 宏量三栏」，用分隔线分区，**不再嵌套子卡**；宏量从独立彩色瓷砖改为 hero 内的细进度条行。
- **半圆仪表（已修 bug）**：`viewBox 0 0 200 104`，arc `M16 96 A84 84 0 0 1 184 96`；中心数字绝对定位于 `top:44px` 居中，**与弧线留足净空、不再被裁切**；已摄入/已消耗下移成独立一行。`stroke-dasharray` 随已摄入/目标动态。
- **导航激活态**：深色图标 + 标签；中心 Co-pilot 黑色圆 + 青柠光环上浮。

## 内容规格

- **餐名**：建议 ≤ 18 字，超出单行省略号；卡片高度不随名称变化（名称区固定）。
- **「为什么」文案**：必须含与用户输入相关的生活要素（作息/独居/做饭意愿/忌口）；纯术语（如"高蛋白低 GI"）视为不合格。建议 40–70 字。
- **依据来源**：一行，前缀 📖，可点击查看出处（RAG 溯源）。
- **AI 总结条**：根据 `cook` / `avoid` 动态拼接。
- **热量**：来自规则引擎，非 LLM 生成（见 PRD P0-5）。

---

## 边界情况

- **空状态**：用户跳过忌口 → 不显示忌口提示；一句生活留空 → 用默认推断，不阻塞。
- **模糊输入**：吐槽词无法匹配关键词 → toast「收到，我按『xxx』调整了一版」并占位，不报错。
- **长文本 / 国际化**：餐名、为什么文案需支持更长字符串，卡片纵向自适应、标题截断。
- **加载**：生成阶段全屏 spinner + 步骤文案；建议真实接口下用骨架屏替代「今日卡片」。
- **错误**：生成失败应提供「重试」按钮（原型未含，需补）。
- **安全拦截**：用户填入疾病/风险信息 → 转为「建议咨询专业人士」，不出计划（PRD P0-5）。

## 无障碍（A11y）

- **焦点顺序**：进度 → 标题 → 选项 chips（从左到右）→ 主 CTA。
- **ARIA**：chip 用 `role="button"` + `aria-pressed`；Segmented 用 `role="tablist"/"tab"` + `aria-selected`；「为什么」按钮 `aria-expanded`；Toast 用 `role="status"`（aria-live polite）。
- **键盘**：所有 chip / 按钮可 Tab 聚焦、Enter/Space 触发；输入框 Enter 提交调整。
- **对比度**：深藏蓝 `ink #16223A` 在奶油底约 14:1（远超 AA）。**注意**：青柠 `lime #A6D938` 是浅色，**按钮文字必须用深蓝 `ink` 而非白色**（原型即如此），白字在青柠上对比不足。橙色 `flame #F47A1F` 火焰数值在白底约 3:1 — 仅用于大号粗体数字，不作正文。
- **触控目标**：所有可点元素 ≥ 44×44pt（chip、发送按钮、导航项均满足）。
- **屏幕阅读播报**：餐卡应朗读「餐次 + 餐名 + 热量」；展开「为什么」时播报理由；调整完成时通过 live region 播报 toast 文案。
- **减少动效**：尊重 `prefers-reduced-motion`，关闭缩放/位移过渡（原型未含，需补）。
