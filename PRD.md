## 0. 一句话定位

> 给健康饮食新手的 **AI 营养教练**：用大白话说一句生活，就拿到一份**懂你、会解释"为什么"**的计划；不会做饭也没关系——材料包送到家，**AI 视频通话一步步教你做**；任何时候有问题，**底部 AI co-pilot 随叫随到**。

v2.0 的产品骨架由三层构成，对应新手从"想吃健康"到"真的做到"的完整链路：
1. **认知 + 规划**：AI 生成 + 解释个性化计划（v1.0 的杀手锏，仍是核心）。
2. **执行**：指定菜谱 → 材料包配送 → **AI 视频教做菜**（回归原始产品诉求：培养做饭习惯）。
3. **坚持 + 答疑**：进度可视化、健康科普、以及贯穿全局的 **AI co-pilot**。

---

## 1. 问题陈述（Problem Statement）

健康饮食新手（典型：独居、三餐外卖、被医生提醒过但无从下手的都市上班族）真正的障碍**不是"不知道健康饮食重要"，而是"知道却做不到、坚持不下来"**。用户研究显示，他们卡在四件事上：**找信息累且无法分辨真假、不会为自己规划、不会做饭/嫌麻烦、照通用模板执行又因不贴合生活而放弃**。

v1.0 解决了前两层（信息与规划）。但"不会做饭、嫌麻烦"是新手从"有计划"到"真执行"之间最大的断点——这正是 v2.0 用**材料包 + AI 视频教学**补上的一环。不解决执行，再好的计划也只是又一张存着不用的食谱。

市场研究：50.7% 用户愿为健康服务付费，最看重**定制化（44%）**与效率；现有竞品（下厨房/小红书/Keep）的共同空白区，正是"千人千面 + 手把手执行"。

## 2. 目标用户

**主目标人群：健康饮食新手 / 小白**（与 v1.0 一致）

代表人物 **Mary**：30 岁，独居租房，工作忙，一日三餐靠外卖，被医生提醒注意饮食，焦虑但不懂术语、讨厌填表、偏好"被告知/被手把手带"。

**四个结构性障碍（产品逐一拆除）**
1. 不懂术语（看不懂 BMI/热量/宏量）
2. 不会表达需求（不知道该问什么）
3. 无法判断真假（分不清哪条信息适合自己）
4. **不会做 / 嫌麻烦**（有计划也落不了地）——v2.0 新增聚焦

**明确不服务（本期）**：减脂进阶、增肌健身、已有成熟方法的"老手"。

## 3. 目标（Goals）

| # | 目标 | 类型 | 衡量方式 |
|---|---|---|---|
| G1 | 打开到拿到第一份可用计划 ≤ 90 秒 | 用户 | 首计划生成时长中位数 |
| G2 | 用户认同"这份计划懂我" | 用户 | 生成后即时反馈 ≥ 70% 选"贴合我" |
| G3 | 用户能说出"为什么这样吃"的理由 | 用户 | "为什么"点开率 ≥ 50% |
| G4 | 提升早期留存 | 业务 | 生成首计划用户次周回访较对照组 +30% |
| G5 | 零健康安全事故 | 合规 | AI 输出违反安全规则逃逸率 = 0 |
| **G6** | **执行闭环跑通：用计划真的做出/吃出一餐** | 用户 | 采用计划后 7 日内完成 ≥ 1 餐打卡或 1 次 AI 教学的比例 ≥ 40% |
| **G7** | **AI co-pilot 成为高频入口** | 用户 | 周活用户中使用 co-pilot ≥ 1 次的比例 ≥ 50% |

> G1–G3 是规划层成败三问（够快/够懂/能学到）；**G6 是 v2.0 新增的执行层北极星**；G5 不可妥协。

## 4. 非目标（Non-Goals）

| 非目标 | 为什么本期不做 |
|---|---|
| 自建社区 / KOL 内容生态 | 易稀释焦点，不是新手最深的坑 |
| 重度游戏化激励体系 | 依赖核心体验先立住，先做轻量连续天数即可 |
| 多目标人群（减脂/增肌）专业配餐 | 配餐逻辑差异大，先把"新手均衡饮食"做透 |
| AI 提供医疗 / 疾病治疗建议 | 合规高风险，明确划在边界外，仅做非医疗饮食引导 |

> **范围变更**：v1.0 把"食材配送闭环"列为非目标；v2.0 将其纳入——但以**材料包 + AI 教做菜**这一执行闭环的形态进入，且为**分阶段、可与第三方供应链合作**的方式，而非自建重供应链（详见阶段规划）。

## 5. 产品结构与信息架构（v2.0 新增）

底部导航 5 项，AI 为中枢：

| 模块 | 角色 | 关键内容 |
|---|---|---|
| **首页** | 主入口 | 今日计划（铺开：采用 → 打勾）+ 进度概览（今日摄入/宏量/连续天数）+ 健康科普 feeds |
| **食谱** | 内容/选择 | 搜索 + 分类 + 食谱卡 → 食物详情（含「材料包已送达 / AI 视频教我做」入口） |
| **✦ AI co-pilot** | **全局中枢** | 底部中心常驻，任意页可唤起；生成/调整计划、拍照记录、营养答疑都在此对话内完成 |
| **进度** | 深度洞察 | 总进度、本周日均、达标天数、连续天数、趋势 |
| **我的** | 账户 | 目标、口味与忌口、材料包订单、提醒、隐私 |

**AI-native 原则**：AI 不是某个页里的输入框，而是贯穿全局的入口；所有"和 AI 说"统一收敛到 co-pilot，feed 里不再散落输入框；科普卡点击即唤起 co-pilot 讲解。

## 6. 需求与验收标准（Requirements）

### P0 · Must-Have（规划层核心，v1.0 延续）

**P0-1 对话式极简输入（问生活，不问数字）**：≤ 3 个对话问题（目标/忌口/做饭意愿）+ 一句生活；其余参数 AI 推断。全程无术语裸露。**非强制**——可全程「跳过」，打开即先看到示范计划。

**P0-2 ≤ 30 秒生成 + 采用闭环**：默认展开"今日三餐"；状态机 `AI 推荐 →（采用今日计划）→ 进行中（打勾）`，主任务唯一清晰。

**P0-3 "人话版为什么"解释**：每餐一条、必须与用户生活挂钩（纯术语判不合格），含可溯源依据。

**P0-4 反应式调整**：自然语言"吐槽即改"（肉太多/不吃早餐/想简单点），不破坏营养约束。

**P0-5 防幻觉混合架构（生死线）**：LLM 负责理解/选品/解释/调整；**热量与宏量计算、份量安全区间、忌口过敏与疾病硬约束由规则引擎/可信库执行**。
- [ ] 安全规则逃逸率 = 0；忌口食物绝不出现；风险信息转建议就医。
- [ ] "为什么"依据可溯源（RAG + 引用）；全局常驻"不构成医疗建议"。

**P0-6 全局 AI co-pilot（v2.0 新增核心）**：底部中心入口，任意页唤起；统一承载生成/调整/拍照/答疑，识别意图后直接操作计划并回到对应页。
- Given 任意主页面；When 点中心 ✦；Then 进入对话，且可一句话完成"重新生成/换一餐/拍照记录/问营养"。
- [ ] 输入框只存在于 co-pilot，不出现在内容 feed 中。

### P1 · 执行层（v2.0 旗舰，回归原始诉求）

**P1-1 材料包 + AI 视频教做菜（旗舰差异点）**
指定菜谱 → 下单材料包 → 送达后从食物详情进入 **AI 视频通话式教学**：教练画面 + 实时字幕 + 分步引导（第 N/M 步）+ 通话控制；结束自动记录这一餐。
- Given 详情页显示「材料包已送达」；When 点「AI 视频教我做」；Then 进入分步教学，可"下一步"推进、末步"完成"后回计划并记录。
- [ ] 步骤、字幕、计时一致；结束写入当日执行记录（支撑 G6）。
- 依赖：材料包供应（建议先与第三方生鲜/净菜供应链合作，不自建）。

**P1-2 拍照记录**：相机识别食物并估算热量，记入对应餐次（作为 co-pilot 内能力 + 入口）。

**P1-3 进度与坚持**：连续天数、今日摄入/宏量、本周日均、达标天数、趋势；轻量连续记录激励。

**P1-4 健康科普 feeds**：首页知识卡，点击唤起 co-pilot 即时讲解（GI、蛋白质摄入、晚餐清淡、外卖技巧等）。

### P2 · Future Considerations

- 社区/KOL、重度游戏化、情绪感知干预、多目标人群配餐、自建供应链、线下活动。
- 架构预留：执行记录、目标类型、供应商对接字段。

## 7. 设计与体验原则（v2.0 新增，沉淀自多轮设计评审）

这些是已在原型中落地、需在开发中保持的硬约束：

**数字单一数据源**：今日卡的热量、宏量、进度环、目标**全部由三餐数据实时计算**——目标 = Σ 未删除餐次热量（默认 1,480）；已摄入 = Σ 已打勾餐次；宏量与之同源。**严禁**任何页面硬编码独立、互相打架的数字（"摄入 0 却显示宏量 160g"这类矛盾不允许出现）。跨页指标（连续 4 天 / 已记录 28 / 达成 40%）必须一致。

**视觉系统（克制 + 专业，Airbnb/Google 取向）**
- 中性底 + **填充卡建立层次**：清新浅绿主卡（深绿字 + 白色宏量瓦片 + 进度环）作锚点，其余白色扁平卡（描边、无大阴影）。
- **绿色只用于 4 处**：浅绿主卡、主 CTA、底部 AI、进度环/条；其余一律中性，避免绿色泛滥。
- **形状统一**：按钮 = 圆角矩形 10px；徽章/筛选 chip = 药丸；二者不混用。功能图标统一线性 SVG（非 emoji）；emoji 仅作食物图片占位。
- **对齐网格**：卡内元素左对齐到 16px；数值列固定宽度对齐；图标与文字垂直居中。
- **文案/时间一致**：问候与时间匹配；术语必带人话解释。

**交互原则**：反应优于表达（先草稿再吐槽）；解释即产品（"为什么"是教育与信任载体）；渐进披露（今天 → 整周）；导航用历史栈，逐层正确回退。

## 8. 核心体验流程

```
【规划】说一句生活(可跳过) → ≤30s 生成 → 看计划 + 每餐"为什么" → 吐槽式调整 → 采用今日计划
        ↓
【执行】某餐「做菜」/ 详情「AI 视频教我做」(材料包已送达) → 分步视频教学 → 完成 → 自动记录、打勾
        ↓
【坚持】进度环/连续天数实时更新 · 科普 feeds 学习
        ↓
【贯穿】任意环节点底部 ✦ AI co-pilot：重新生成 / 换一餐 / 拍照记录 / 问营养
   （底层：规则引擎兜营养与安全，LLM 负责懂你、解释与对话）
```

## 9. 成功指标（Success Metrics）

**先导（数天～数周）**：首计划生成完成率 ≥ 80%；生成时长中位数 ≤ 30s；"懂我"认同率 ≥ 70%；"为什么"点开率 ≥ 50%；反应式调整使用率 ≥ 40%；**采用计划率**、**AI 教学/打卡完成率 ≥ 40%（G6）**、**co-pilot 使用率 ≥ 50%（G7）**。

**滞后（数周～数月）**：次周回访 +30%；4 周留存（建基线对比）；材料包复购率（执行层商业验证）；NPS；安全事故 = 0。

## 11. 阶段规划（Timeline / Phasing）

**Phase 0 · 验证"懂我"（不写代码，1–2 周）**：绿野仙踪测试 5 位 Mary 型用户，验证 G2。不过关不启动。

**Phase 1 · 规划层 MVP（P0 全集）**：对话输入 → 生成 → 解释 → 调整 → 采用闭环 → 防幻觉架构 + 全局 co-pilot。达成 G1–G3、G5、G7。

**Phase 2 · 执行层旗舰（P1-1 为主）**：先在 1 个城市与第三方净菜供应商合作试点材料包；AI 视频教做菜首版可用"预录步骤 + AI 字幕/语音"降本，验证 G6 与材料包复购。配套拍照记录、进度、科普。

**Phase 3 · 规模化与深化（P2）**：依数据决定供应链投入、激励体系、多人群扩展。

**依赖**：可信营养库/规则引擎（P0-5 前置）；材料包供应合作（P1-1 前置）；合规文案过审（上线前置）。

## 12. 一句话风险提醒

两个不可塌的支点：**① "懂我"假设（Phase 0）**、**② P0-5 安全边界**。v2.0 新增第三个需尽早验证的支点——**③ 材料包 + AI 教学的履约成本与复购**：执行闭环是差异化关键，但也是最重的一环，务必小范围、第三方、低成本先验证，再决定是否加码。
